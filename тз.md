## 1. О проекте

**PPR Travel** — социальное веб-приложение для путешественников на русском языке.

Пользователи могут:
- отмечать посещённые места на интерактивной карте мира;
- создавать гео-метки и маршруты;
- делиться маршрутами с другими;
- следить за активностью друзей;
- участвовать в travel-событиях.

**Структура репозитория:**
- `frontend/` — Vue 3 SPA (порт 5173)
- `backend/` — Express API (порт 3000)

**Текущая стратегия разработки:** бэкенд содержит только аутентификацию; весь остальной функционал реализован на фронтенде и работает через **моковые данные** (`VITE_USE_MOCKS=true`).

## 2. Технологический стек

### Frontend
| Компонент | Технология |
|-----------|------------|
| Фреймворк | Vue 3 (Composition API, `<script setup>`) |
| Язык | TypeScript |
| Сборка | Vite 5 |
| Маршрутизация | Vue Router 4 |
| Состояние | Pinia |
| Утилиты | VueUse (`useDebounceFn` и др.) |
| HTTP | Axios (JWT + refresh cookie) |
| Карты | Leaflet, world-atlas, topojson-client |
| Стили | SCSS (RSCSS-подход) |

### Backend
| Компонент | Технология |
|-----------|------------|
| Runtime | Node.js + Express 4 |
| Язык | TypeScript |
| БД | MongoDB (Mongoose 8) |
| Аутентификация | JWT + httpOnly refresh cookie, bcrypt |
| Валидация | Zod |

### Переменные окружения (frontend/.env)
| Переменная | Значение | Назначение |
|------------|----------|------------|
| `VITE_API_URL` | `http://localhost:3000/api` | Базовый URL API |
| `VITE_USE_MOCKS` | `true` | Включить моковые данные вместо API |
| `VITE_YANDEX_MAPS_API_KEY` | *(пусто)* | API-ключ JavaScript API Яндекс Карт v3 |
| `VITE_YANDEX_MAPS_LANG` | `ru_RU` | Язык карты (опционально) |

### Карта — Yandex Maps v3 (`config/yandex-maps.ts`)
- Загрузка скрипта `https://api-maps.yandex.ru/v3/` с ключом `VITE_YANDEX_MAPS_API_KEY`
- Язык карты: `VITE_YANDEX_MAPS_LANG` (по умолчанию `ru_RU`)
- Провайдер карты: `map-provider/yandex-map.ts` — используется в `MapPage`, `RouteDetailPage`, `VisitedMapView`, `GeoMarksMapView`

## 6. Функционал по модулям

**Реализовано и работает (бэкенд + mock-режим):**
- Регистрация (email, пароль, имя)
- Вход / выход
- JWT access token + httpOnly refresh cookie
- Автообновление токена при 401
- Восстановление сессии при загрузке приложения (`GET /api/auth/me`)

**Реализовано на frontend (mock-режим / UI без бэкенда):**
- Редактирование профиля (имя, дата рождения, город, bio, интересы)
- Загрузка аватара и обложки
- Смена пароля
- Настройки приватности (`public` | `friends`)
- Email-уведомления (лайки, заявки в друзья)

**Удалено из UI:**
- Виджет «Оцени страницу» (1–5 звёзд) — API-методы в `authApi` остались, UI убран

### 6.2 Карта (#картаППР)

**Вкладки (2 шт.):**
1. **Все метки** — все посещённые места всех пользователей, подсветка стран (choropleth)
2. **Мои метки** — фильтрация тех же данных по текущему пользователю

> Ранее были вкладки «Мои метки» (гео-метки) и «Хочу» (wishlist) — **убраны с карты**. Компоненты `GeoMarksMapView.vue` и `WishlistView.vue` остались в кодовой базе, но не подключены.

**Архитектура:**
- Единый компонент `VisitedMapView` с пропом `mode: 'all' | 'my'`
- Один набор данных; разница только в фильтрации по `userId`
- Query-параметр `?tab=my` переключает на вкладку «Мои метки»

**Возможности:**
- Клик по карте → форма «Я здесь был!» (**по центру экрана** по горизонтали)
- Редактирование и удаление своих мест
- Лайки на местах
- Хороплет стран по статистике (`countryCodes`)
- Русскоязычные тайлы карты (при наличии MapTiler-ключа)

**UX-исправления:**
- z-index вкладок карты ниже выпадающих меню шапки (не перекрывают профиль/уведомления)
